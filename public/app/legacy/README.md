# legacy/（仅构建产物 / 不维护）

此目录下的文件属于历史遗留或构建/拆分后的产物（legacy/split artifact）。

- 请不要在此目录内进行功能修改、Bug 修复或新增功能。
- 这些文件通常不会被运行时直接加载，或会在后续的构建/拆分过程中被重新生成并覆盖。

## 正确的修改入口

运行时加载顺序由 `public/app.js` 中的 `scripts` 数组决定。

请在以下目录进行实际开发与修改：

- `public/app/core/`：全局状态、启动流程、工具等
- `public/app/services/`：存储、翻译、网络等服务
- `public/app/features/`：功能模块（文件导入、翻译、术语库、项目管理等）
- `public/app/ui/`：UI 组件与事件监听

## 为什么保留 legacy/

- 用于对照旧实现/调试排查
- 部分环境下可能被工具链引用或作为拆分产物缓存

如果你确实需要删除该目录，请先确认没有脚本/构建流程依赖它。
