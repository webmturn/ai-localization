帮助文档

快速开始

1. 导入文件

点击上方「浏览文件」或拖拽文件到上传区域。支持 XML、Android strings.xml、XLIFF（.xlf/.xliff）、JSON、YAML、CSV/TSV、PO、RESX、iOS Strings、Qt TS 等格式；其他扩展名会按文本方式尝试解析。

2. 查看翻译项

解析后翻译项显示在中间面板，左侧为原文、右侧为译文。可使用搜索、筛选和分页快速定位。

3. 执行翻译

选中需要翻译的项，点击「翻译选中」或「翻译全部」。使用在线引擎（OpenAI / DeepSeek / Google）时，请先在「设置」→「翻译」中配置对应 API Key。

4. 导出结果

翻译完成后，在右侧面板点击「导出翻译」并选择格式。首次导出会按需加载组件，可能稍慢一次，之后会恢复正常。

常见问题

Q: 支持哪些文件格式？

A: 支持 XML、Android strings.xml、iOS Strings、JSON、YAML、CSV/TSV、XLIFF、PO、RESX、Qt TS 等。其他扩展名会按文本方式尝试解析。

Q: 如何配置 API 密钥？

A: 打开「设置」→「翻译」，在所选引擎下填写 API Key，会自动保存。所选引擎未配置或无效时，批量翻译会中止并提示。

Q: 为什么第一次导出/术语导入导出/质量检查会慢？

A: 为加快首屏加载，相关模块和 Chart.js、SheetJS 为按需加载，首次使用会加载一次，之后会正常。

Q: 数据存储在哪里？

A: 项目与设置等数据均保存在浏览器本地（localStorage + IndexedDB），不会上传到任何服务器。自动保存功能默认每10秒检查并保存更改。

Q: 如何暂停/恢复自动保存？

A: 可在控制台使用 autoSaveManager.pause() 暂停，autoSaveManager.resume() 恢复。

Q: 质量检查包含哪些项目？

A: 包括空译文、长度异常、占位符缺失、数字不一致、标点符号异常、重复词、HTML标签不匹配、未翻译检测等。

开发与部署

入口与本地使用

入口为 public/index.html，浏览器打开即可使用。支持 file:// 直接打开；若遇网络或跨域限制，可换浏览器或调整网络/代理。

样式构建

样式由 Tailwind 构建，输出到 public/styles.css。常用：npm run build-css 生成样式，npm run watch-css 开发时监听。

第三方库

资源在 public/lib/，可用 npm run update-cdn 更新。Chart.js、SheetJS 等为按需加载。

部署

仅需发布 public/ 目录即可。
